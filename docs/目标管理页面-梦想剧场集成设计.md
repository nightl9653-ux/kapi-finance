## 目标管理页面-梦想剧场集成设计

### 0. 背景与目标
- 本平台已存在“目标管理”页面（实际实现位于 `src/app/[locale]/goals/page.tsx`），展示用户创建的财务目标，并提供创建、编辑、删除能力。
- 需要为每一个目标增加增强模块“梦想剧场”（名称可调整），用于基于目标信息与用户补充关键词生成：
  - AI 撰写的“梦想小作文”（中文/英文）
  - 一段无文字的视频画面（视频画面与语言无关）
  - 随语言切换变化的旁白音频（TTS）与字幕（SRT）
- 关键约束：所有 AI/视频/TTS/字幕调用必须在服务端完成，避免泄露密钥；模块内语言切换不影响全局 next-intl 语言。

### 1. 现状假设（以当前仓库为准）
- 路由与页面形态
  - 当前“目标管理”页路由形态为 `/{locale}/goals`，页面为 Server Component。
  - 页面同时承担“概览 + 列表 + 每个目标的编辑面板（锚点展开）”的职责。
- 数据来源与表
  - 当前页面读取与写入的表名为 `financial_goals`，并且按 `user_id` 过滤。
  - 目标字段形态与示例类似：id、user_id、name、type、target_amount、current_amount（相当于 saved_amount）、deadline、priority、created_at 等。
- 认证方式
  - 使用 Supabase Auth；服务端通过 `supabase.auth.getUser()` 判断登录态并做重定向。
- UI 与国际化
  - UI 使用 shadcn 风格组件（按钮、输入框、标签等）。
  - next-intl 用于页面级文案翻译（`getTranslations`）。

### 2. 总体方案（推荐 A：视频绑定“基准故事”，语言产物独立存储）
为避免“切换语言导致 story_id 与视频绑定关系混乱”，采用以下语义：
- 基准故事（Base Story）：同一次“生成小作文”形成一个基准故事版本，用一个 `story_id` 表示，携带 keywords 与 freeText 等上下文。
- 视频画面：只与基准故事绑定；切换语言不改变视频画面 URL。
- 多语言文本、旁白音频、字幕：与 (story_id, locale) 绑定，作为“语言产物”单独缓存与复用。

对外表现：
- 用户生成一次故事后，可生成一次视频画面；之后语言切换只切换文本/音频/字幕，不会触发新的视频画面生成。
- 用户修改关键词重新生成故事时，形成新的 story_id；此时可让用户选择“复用上一版画面”或“生成新画面”。

### 3. 前端模块位置与交互（以当前页面结构落地）
#### 3.1 插入点建议
当前 `/{locale}/goals` 的目标列表中，每个目标项包含：
- 顶部摘要（名称/类型/金额/截止日期/优先级）
- 操作区（编辑锚点、删除）
- 展开的编辑面板（表单更新目标）

建议将“梦想剧场”作为该目标项的子区域，放在编辑面板下方，结构为：
- 目标项卡片
  - 摘要区
  - 编辑面板
  - 梦想剧场折叠卡片（默认收起）

理由：
- 用户在同一个目标上下文内，编辑字段后可以立即生成对应的梦想内容。
- 不需要引入新的详情页路由即可完成第一阶段交付。

#### 3.2 交互形态
- 折叠卡片标题（例如“梦想剧场”），右侧提供模块内语言切换开关（zh/en）。
- 初始状态（该目标尚未生成过基准故事）
  - 关键词选择（按目标类型动态提供 8–10 个预设关键词）
  - 自定义关键词输入（逗号分隔或逐个添加）
  - 自由文本补充（选填）
  - “生成梦想小作文”按钮
- 已生成状态（存在基准 story_id）
  - 显示当前 moduleLocale 的小作文文本
  - 显示视频预览区（若已生成视频则展示；否则展示“生成视频”按钮）
  - 显示旁白与字幕状态（可播放/可切换语言；失败则降级为仅文本）
  - “重新生成”入口（关键词变化时）
  - “复用画面”策略提示（当重新生成故事时出现）

#### 3.3 模块内语言切换策略
- 模块内维护 `moduleLocale`，默认读取页面 locale 作为初始值。
- 切换 `moduleLocale` 不触发路由语言切换，不影响页面其他文案。
- 切换时：
  - 若存在基准 story_id：请求生成/读取该 locale 的语言产物（文本、音频、字幕），并热更新 UI。
  - 若不存在基准 story_id：提示先生成故事，或直接以新语言生成基准故事（产品策略二选一，建议提示先生成，避免无上下文造成体验不一致）。

### 5. 三个核心 Server Action 定义（自然语言）
以下三者是“梦想剧场”的最小闭环核心；其他如日志、任务轮询属于配套设施。

#### 5.1 generateStoryForGoal（生成基准故事）
- 输入参数
  - goalId：目标 ID
  - selectedKeywords：预设关键词集合
  - customKeywords：自定义关键词集合（可为空）
  - freeText：补充描述（可为空）
  - locale：zh 或 en
- 服务端读取
  - 从 `financial_goals` 读取该 goalId 的目标详情（name/type/target_amount/current_amount/deadline/priority）
  - 从会话读取当前 user，并校验该目标归属
- 主要行为
  - 调用 LLM（OpenAI SDK，baseURL 指向 DMXAPI，模型 gpt-4o）
  - 系统提示词核心要点
    - 你是理财生活规划师
    - 用户为该目标做了规划与储蓄，结合目标类型与金额与进度，写实现后的生活场景
    - 肯定努力，画面感强，结尾轻对比“无规划的可能”
    - 250–400 字，语言由 locale 决定
  - 将关键词归一化（去空格、去重、排序；英文统一大小写策略），用于缓存与去重
- 输出结果
  - storyId（基准故事 ID）
  - content（生成的小作文文本，语言为 locale）
  - locale
- 副作用
  - 写入 `goal_stories` 一条记录（见第 6 节）
  - 写入 AI 调用日志（耗时、是否命中缓存、错误等）

#### 5.2 generateVideoForStory（为基准故事生成视频画面）
- 输入参数
  - storyId：基准故事 ID
  - reusePolicy（可选）：是否允许复用旧画面（默认允许，具体策略由产品决定）
- 服务端读取
  - 从 `goal_stories` 取 keywords/freeText/goal_id，必要时可连同目标信息一起取
- 主要行为
  - 先生成一个不超过 25 词的英文视频 prompt，并强制追加约束：
    - no text
    - no letters
    - no watermark
  - 调用 DMXAPI 视频接口（Kling/Seedance）提交异步任务，获得 taskId
  - 在 `goal_videos` 中创建或更新记录，状态置为 processing
- 输出结果
  - videoRecordId
  - status（processing）
- 副作用
  - 启动异步任务结果回收机制（见第 7 节），成功后回写 video_url 与 status
  - 若失败，记录失败原因并触发降级策略（返回预置视频 URL 或前端按目标类型选择预置视频）

#### 5.3 generateLocalizedStoryMedia（生成或读取语言产物：文本/音频/字幕）
- 输入参数
  - storyId：基准故事 ID
  - newLocale：目标语言（zh 或 en）
- 服务端读取
  - 从会话读取当前 user，并校验 storyId 归属
  - 查询是否已有 (storyId, newLocale) 的语言产物记录
- 主要行为
  - 若已存在：直接返回对应文本/音频/字幕 URL
  - 若不存在：
    - 以“翻译并轻微本地化”为默认策略生成新语言文本，保持镜头语义一致，避免完全重写带来画面不匹配
    - 生成 TTS 音频（建议使用 gpt-4o-mini-tts 或等价 TTS），上传至 Supabase Storage
    - 将文本切分为短句并生成 SRT 字幕，上传 Storage
    - 写入语言产物记录并返回
- 输出结果
  - localizedContent（新语言文本）
  - audioUrl（旁白音频）
  - subtitleUrl（字幕）
  - locale（newLocale）
- 副作用
  - 写入语言产物表（见第 6 节）
  - 错误时允许降级：只返回文本，音频字幕失败给出可读错误信息

### 6. Supabase 数据模型扩展（适配当前平台表名）
说明：当前目标表实际为 `financial_goals`，下述新增表通过外键与其关联。

#### 6.1 goal_stories（新增，存基准故事与生成上下文）
- id：主键
- goal_id：外键 -> financial_goals.id
- keywords：jsonb（归一化后的关键词数组或结构）
- free_text：text
- locale：text（本条基准故事的生成语言）
- content：text（本条基准故事文本）
- created_at：timestamp
- 建议约束（用于缓存去重）
  - 同一 goal_id 下，对“关键词归一化结果 + free_text + locale”的组合做唯一或准唯一约束（可用 hash 字段实现），避免重复付费生成

#### 6.2 goal_videos（新增，存视频画面，绑定基准 story）
- id：主键
- story_id：外键 -> goal_stories.id
- video_url：text
- status：text（processing / succeeded / failed）
- provider_task_id：text（可选，用于追踪 DMXAPI taskId）
- created_at：timestamp
- 建议约束
  - story_id 唯一（每个基准故事最多一个视频画面）

#### 6.3 goal_media（新增，存不同语言的文本/音频/字幕产物）
- story_id：外键 -> goal_stories.id
- locale：text
- content：text（建议在此存“该 locale 的文本版本”，避免为语言再建一条 goal_stories）
- audio_url：text
- subtitle_url：text
- created_at：timestamp
- 建议约束
  - (story_id, locale) 唯一，保证语言切换直接复用

#### 6.4 日志表（建议新增）
用于可观测性与成本控制（不属于核心链路，但建议尽早加入）。
- ai_logs 或 error_logs
  - 记录：user_id、goal_id、story_id、action_type（story/video/tts/subtitle/translate）、duration_ms、status、error_message、cache_hit 等

### 7. 异步视频任务回收（Edge Function 或 Cron）
视频生成是异步的，需要一个稳定的“任务回收”机制：
- 提交视频任务后写入 `goal_videos` 的 provider_task_id 与 processing 状态。
- 回收器定期扫描 processing 任务并向 DMXAPI 查询状态：
  - 成功：写入 video_url，状态改为 succeeded
  - 失败：状态改为 failed，并记录错误原因；前端触发降级逻辑
- 建议限制
  - 轮询频率上限与最大等待时间（例如 60–120 秒）以及重试次数，避免成本与资源失控

### 8. 模块内语言切换时序（文本示意）
- 用户点击梦想剧场内部语言切换（例如 zh -> en）
- 前端更新 moduleLocale，进入 loading 状态
- 若无 storyId：提示先生成基准故事，流程结束
- 若有 storyId：
  - 前端请求 generateLocalizedStoryMedia(storyId, newLocale)
  - 服务端校验权限后：
    - 若命中缓存：直接返回 content/audio/subtitle
    - 若未命中：生成翻译文本 -> 生成 TTS -> 生成 SRT -> 上传 Storage -> 写入 goal_media -> 返回
  - 前端收到结果：
    - 文本切换为新语言
    - 视频画面保持不变（仍使用 storyId 对应的 video_url）
    - 音频与字幕切换到新语言 URL
  - 若音频或字幕失败：显示错误提示并降级为仅文本

### 9. 与现有平台的集成检查点
- 登录与权限
  - 所有 Server Action 必须基于服务端会话确认 user，且通过外键链路校验该 goal/story 归属。
- RLS
  - financial_goals：仅 owner 可读写
  - goal_stories / goal_videos / goal_media：仅能访问属于本人 financial_goals 的记录（通过外键与策略实现）
- UI 一致性
  - 折叠卡片、按钮、输入、多选、提示等复用现有 shadcn 组件风格，保证视觉一致。
- next-intl 边界
  - 模块内切换不触发路由语言变更，避免影响页面其他模块文案。
- 成本控制
  - keywords 归一化与去重缓存必须是第一阶段就具备的能力。
  - 视频画面复用是二阶段重点能力，需设计为可选策略而非隐藏逻辑。

### 10. 测试场景（覆盖关键路径）
- 首次为目标生成小作文与视频
  - 选择关键词与补充描述，生成基准故事成功入库并展示
  - 触发视频生成进入 processing，最终出现可播放 video_url
  - 若视频超时或失败：提示并降级为预置视频
- 同一目标下切换语言，验证视频画面不变
  - 切换 moduleLocale 后：
    - 文本切换正确
    - 音频与字幕更新为新语言
    - video_url 不变化
- 修改关键词后重新生成，验证旧视频复用或新生成
  - 新的关键词触发新的基准 story
  - 用户选择复用旧画面或生成新画面，两条路径均可正确完成
- 视频接口失败时的降级行为
  - DMXAPI 返回失败或长时间无结果
  - 记录 failed 状态与错误信息
  - 前端切换到预置视频并允许继续生成/切换旁白字幕

### 11. 交付节奏建议（两阶段，降低集成风险）
- 阶段一（先稳住“故事 + 语言切换”）
  - DreamTheater UI（折叠卡片、关键词表单、文本展示、模块内语言切换）
  - generateStoryForGoal 与 generateLocalizedStoryMedia（文本优先，音频字幕可在阶段一后半加入）
  - goal_stories 与 goal_media（含去重与缓存）
  - ai_logs/error_logs
- 阶段二（补齐“视频画面 + 异步链路”）
  - generateVideoForStory
  - goal_videos
  - 异步任务回收器（Edge Function 或 Cron）
  - 预置视频降级策略与“复用画面”产品交互
